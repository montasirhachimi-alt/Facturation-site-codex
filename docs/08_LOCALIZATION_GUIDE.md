# HicoPilot Localization Guide

## Purpose

This guide defines the official French terminology for visible HicoPilot UI. Every new screen, action, button, empty state and navigation item must follow this vocabulary.

## Language Rule

HicoPilot is French-first.

Visible UI must not mix French and English unless the term is an accepted product or technical brand term such as CRM, PDF, Excel, Email, IA or Pipeline.

## Button Conventions

Buttons must be contextual.

Use:

- `Nouvelle société`
- `Nouveau contact`
- `Nouvelle tâche`
- `Nouvelle note`
- `Planifier une réunion`
- `Enregistrer`
- `Archiver`

Avoid generic labels:

- `Nouvelle entrée`
- `Créer`
- `Ajouter`
- `New`
- `Create`

## Page Title Conventions

Page titles should describe the user-facing workspace.

Use:

- `Centre de relation client`
- `Sociétés`
- `Clients`
- `Vue d'ensemble CRM`
- `Opportunités`

Avoid:

- `Module ERP`
- `Module`
- `Page`
- `Dashboard`

## Empty State Conventions

Every empty state must explain:

1. What the section represents.
2. Why it is empty.
3. What the user should do next.

Example:

> Aucune réunion planifiée. Planifiez une réunion depuis une fiche contact.

## Badges And Statuses

Use French labels for statuses:

- `Actif`
- `Inactive`
- `Archivée`
- `Ouverte`
- `En cours`
- `En attente`
- `Terminée`
- `Annulée`
- `Gagnée`
- `Perdue`

## Accepted Terms

The following terms may remain as-is:

- CRM
- PDF
- Excel
- Email
- IA
- Pipeline
- HicoPilot

## Review Checklist

Before completing a UI sprint:

- Sidebar labels are French.
- Topbar labels are French.
- Buttons are contextual.
- Empty states are useful and French.
- Tables use French column labels.
- Tabs use French labels.
- No generic `Module ERP` copy remains.
- No mixed English/French UI remains in the touched area.
