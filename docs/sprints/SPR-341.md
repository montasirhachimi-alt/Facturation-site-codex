# SPR-341 — Inline Entity Creation

Date: 2026-07-09

## Objective

Extend Smart Entity Picker so users can create a missing related entity directly inside the current form workflow.

## Scope

- UI and product-experience layer only.
- Local/demo creation behavior only.
- No Runtime, Services, APIs, Prisma, database, permissions or business logic changes.
- No backend endpoints, AI, persistence or workflow automation.

## Inline Creation Architecture

- `SmartEntityPicker` now supports optional inline creation through props.
- The generic picker remains module-agnostic and does not import CRM or Sales modules.
- Business-specific picker items are still created by the consuming form through `onCreate`.
- Created items are stored in picker-local state and selected immediately.
- Existing form values remain string-compatible, preserving current submit behavior.

## Picker API Changes

Added optional props:

- `allowCreate`
- `createLabel`
- `entityType`
- `initialCreateValue`
- `onCreate`

When creation is disabled, picker behavior is unchanged.

## Supported Scenarios

Enabled inline creation for:

- Quote dialog customer picker.
- Quote dialog company picker.
- Quote dialog contact picker.
- Customer dialog company picker.
- Invoice Quick Create customer picker.

Not enabled for:

- Payment Quick Create invoice picker, because creating an invoice inline from payment would be a fragile workflow without a stable invoice creation dialog.
- Quote source picker, because creating a quote inline from invoice creation would duplicate the quote workflow.

## UX Behavior

- When search has no results and creation is enabled, the picker shows `+ Créer “query”`.
- Arrow keys can focus the create action.
- Enter opens a compact nested creation panel.
- Esc closes the nested creation panel without closing the parent form.
- Saving creates a local picker item, selects it, closes the panel and returns focus to the original picker.
- Parent form state is preserved.

## Desktop Result

Users can keep working inside the current dialog when a related customer, company or contact does not exist yet. The interaction remains compact and close to the field that triggered it.

## Mobile Result

The inline panel stays inside the picker dropdown, uses full-width controls and avoids horizontal overflow. Buttons remain comfortable for touch.
