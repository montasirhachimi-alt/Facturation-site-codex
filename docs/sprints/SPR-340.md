# SPR-340 — Smart Entity Picker

Date: 2026-07-09

## Objective

Create a reusable Smart Entity Picker so related business objects can be selected by typing a few letters instead of using classic static selects.

## Scope

- UI and product-experience layer only.
- Local filtering only.
- Existing fixtures and module data only.
- No Runtime, Services, APIs, Prisma, database, permissions or business logic changes.
- No backend search, AI, persistence or new workflow automation.

## Smart Entity Picker Architecture

- Added a reusable `SmartEntityPicker` form control under `src/ui/forms/`.
- Added shared picker types for reusable entity result contracts.
- Added picker utilities for local normalization, scoring and filtering.
- Added an entity picker registry/adapters file that maps existing local fixtures into picker items.
- Kept data adapters out of the broad `@/ui` barrel to avoid forcing every UI import to load business module seed data.

## Supported Entity Types

V1 supports local picker items for:

- Companies
- Contacts
- Customers
- Quotes
- Invoices

Products are prepared at the registry level as a disabled entity type because no clean reusable local product module/data source exists in this workspace.

## Data Sources Used

- `crmCompanySeed`
- `crmContactSeed`
- `crmCustomerSeed`
- `quoteSeed`
- `invoiceSeed`

No new large datasets were duplicated or invented.

## Forms Integrated

- Quote creation dialog:
  - Client picker
  - Company picker
  - Contact picker
- Quick Create opportunity surface:
  - Company picker
  - Contact picker
- Quick Create invoice surface:
  - Customer picker
  - Quote picker
- Quick Create payment surface:
  - Invoice picker
- Customer dialog:
  - Company-name field now supports searchable company selection while preserving the string value expected by the existing form.

Integrations were limited to stable existing dialogs and Quick Create surfaces. Contact/company selection was not forced into dialogs that do not expose a stable related-entity field.

## Picker UX

- Type-to-search local filtering.
- Keyboard navigation with Arrow Up and Arrow Down.
- Enter selects the active result.
- Esc closes the picker.
- Click selects a result.
- Outside click closes the picker.
- Clear button removes the selected value.
- Empty state explains when no result matches.
- Result rows show icon, title, entity badge and metadata.

## Accessibility

- Uses combobox/listbox semantics.
- Exposes expanded state and active descendant.
- Keeps visible focus state through the existing BOSIACO form ring.
- Does not trap Tab.

## Desktop Result

Data entry now feels closer to the Command Center: compact, searchable and fast. Users can find a company, customer, contact, quote or invoice without scanning a long select list.

## Mobile Result

The picker remains full-width inside form sections, uses readable input sizing and keeps result rows large enough for touch selection without horizontal overflow.
