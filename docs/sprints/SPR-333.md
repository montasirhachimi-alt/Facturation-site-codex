# SPR-333 — CRM Workspace Independence

## Executive Summary

SPR-333 restores predictable CRM navigation. Contacts, Activités, Réunions, Tâches and Notes now open their own CRM workspaces instead of redirecting users toward Sociétés or an example contact.

The sprint preserves contextual relationships: companies still link to contacts, contacts still link back to companies, and activity, meeting, task and note rows keep direct links to their related CRM entities.

No Runtime, Services, APIs, Prisma, database, permissions or business logic were changed.

## What Changed

- Updated CRM navigation so every sidebar item points to its corresponding workspace.
- Added independent CRM workspace routes for:
  - `/crm/contacts`
  - `/crm/activities`
  - `/crm/meetings`
  - `/crm/tasks`
  - `/crm/notes`
- Added shared contextual CRM workspace UI that reuses existing CRM seeds and domain types.
- Updated CRM route definitions so Meetings and Tasks are official CRM routes.
- Adjusted CRM Home and Dashboard shortcuts that previously pointed to `/crm` or a contact example.
- Reworded CRM Home helper copy so it no longer teaches users that contextual objects must be opened through another parent workspace.

## Navigation Decisions

| Item | Before | After |
| --- | --- | --- |
| Contacts | Redirected to Sociétés | Opens `/crm/contacts` |
| Activités / Timeline | Redirected to Sociétés | Opens `/crm/activities` |
| Réunions | Opened an example contact | Opens `/crm/meetings` |
| Tâches | Opened an example contact | Opens `/crm/tasks` |
| Notes | Opened an example contact | Opens `/crm/notes` |

## Context Preserved

- Contact rows link to the contact detail and related company.
- Activity rows link to the related contact when available, otherwise the company.
- Meeting rows link to the primary contact and related company.
- Task rows link to the assigned contact and related company.
- Note rows link to their contact or company context.
