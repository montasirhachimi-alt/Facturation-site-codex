# SPR-326 — Global Search Foundation

## Summary

SPR-326 introduces the UI foundation for a modern global search entry point in BOSIACO/HicoPilot. The sprint does not add AI, backend search, database indexing or fuzzy search. It prepares a calm, fast-feeling shell that future sprints can connect to Global Search, Command Palette, AI Assistant and Quick Navigation providers.

## Objective

Make navigation faster as the product grows by creating one premium search entry point in the top application header.

## Scope

- Added an input-like global search button to the topbar.
- Added platform-aware shortcut display: `⌘K` on Mac and `Ctrl+K` elsewhere.
- Reworked the platform search dialog as a UI-only foundation.
- Prepared static sections: Recent, Suggestions and Navigation.
- Added keyboard support for shortcut open, ESC close, arrow navigation and Enter selection.
- Added outside-click close and focus trapping.
- Added a resolver contract so future providers can inject sections later.

## Product UX Decisions

- The topbar search looks like an input because search is a primary navigation affordance, but it remains a button so users understand it opens a command-style dialog.
- The dialog uses generous spacing, rounded corners, soft shadow and a blurred overlay to feel central without being visually loud.
- Sections are visible from day one so the product teaches the future mental model before real search providers are connected.
- Static placeholder content keeps the experience polished while avoiding fake backend search behavior.
- Keyboard behavior is treated as part of the foundation, not as a later enhancement.

## Architecture Decisions

- Kept React search UI inside `src/platform/search/`, following ADR-009.
- Introduced `UniversalSearchSectionResolver` as the future injection point for search providers.
- Removed command/search service wiring from the ERP shell for this sprint so the implementation remains UI-foundation only.
- Kept Core, Services, Runtime, Prisma, database, APIs, permissions and business logic untouched.
- `src/navigation/` was reviewed as requested, but it does not exist in the current repository; navigation remains module/service-driven.
