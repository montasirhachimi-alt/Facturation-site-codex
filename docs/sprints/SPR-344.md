# SPR-344 — Keyboard Everywhere

Date: 2026-07-10

## Objective

Create a reusable keyboard foundation for high-frequency BOSIACO workflows without changing Runtime, Services, APIs, Prisma, database schema, permissions, authentication or business logic.

## Scope

- Product experience and accessibility only.
- Existing Command Center remains the single `Cmd/Ctrl + K` surface.
- Existing dialogs, form validation and local creation flows remain the source of truth.
- No backend work, AI, workflow automation or destructive shortcut behavior was introduced.

## Keyboard Architecture

Added `src/platform/keyboard/` as a client-safe platform layer:

- `keyboard-shortcut.types.ts`
- `keyboard-shortcut.utils.ts`
- `use-keyboard-shortcut.ts`
- `use-workspace-create-shortcut.ts`
- `use-table-keyboard-navigation.ts`
- `keyboard-shortcut-provider.tsx`
- `keyboard-shortcuts-help.tsx`
- `index.ts`

The platform contains only dependency-light keyboard types, utilities, hooks and help UI. It does not import CRM, Sales, Runtime, Services or module pages.

## Implemented Shortcuts

Global:

- `Cmd/Ctrl + K`: opens the existing Command Center.
- `Cmd/Ctrl + N`: opens the current workspace creation flow when that workspace has a stable creation dialog.
- `?`: opens Keyboard Shortcuts Help when focus is not inside an editable field.

Forms:

- `Cmd/Ctrl + Enter`: submits stable entity dialogs through the existing form submit path.
- `Cmd/Ctrl + S`: submits stable entity dialogs through the same path and prevents browser Save Page only when handled.

Tables/lists:

- `Arrow Down`: next visible row.
- `Arrow Up`: previous visible row.
- `Home`: first visible row.
- `End`: last visible row.
- `Enter`: opens the active row where a stable detail route exists.
- `Space`: toggles row selection where row selection already exists.

Contextual actions:

- `Arrow Left / Arrow Right`: move between action chips.
- `Enter / Space`: activate focused action chips while preserving native link/button semantics.

Dialogs/transient surfaces:

- `Escape`: closes the active picker, inline creation panel, PDF preview, dialog or shortcut help surface one layer at a time.

## Scope And Conflict Resolution

V1 uses component ownership rather than a heavy central conflict engine:

- Command Center keeps its existing single global listener for `Cmd/Ctrl + K`.
- Keyboard Shortcut Provider owns only `?` and the help surface.
- Workspace pages opt into `Cmd/Ctrl + N`.
- Dialogs own form submit shortcuts while open.
- Tables own row navigation only when focus is within the table area.
- Pickers stop propagation for picker-level Escape/Enter/Arrow behavior so parent dialogs do not close accidentally.

This keeps one handler responsible for each supported surface and avoids duplicate Command Center listeners.

## Editable Target Safety

Added shared utilities:

- `isEditableTarget`
- `shouldIgnoreKeyboardEvent`
- `isModKey`
- `matchesShortcut`
- platform shortcut label helpers

Global shortcuts are ignored when focus is inside inputs, textareas, selects, contenteditable areas, textbox roles or future `data-keyboard-editable` surfaces.

## Workspace Create Integrations

`Cmd/Ctrl + N` is enabled on stable workspaces with existing dialogs:

- Quotes workspace → New Quote.
- Invoices workspace → New Invoice.
- Customers workspace → New Customer.
- Companies workspace → New Company.
- Company Contacts workspace → New Contact.

Payments and Opportunities do not receive standalone creation shortcuts in V1 because they do not yet have stable standalone creation dialogs. Payments continue to be recorded from Invoice details, and Opportunities continue to be created from existing CRM context.

## Form Integrations

The shared `EntityDialog` now supports:

- `Cmd/Ctrl + Enter`
- `Cmd/Ctrl + S`
- `Escape`

Submission uses `requestSubmit()` on the active form, preserving:

- existing validation
- Quote and Invoice line-item validation
- Smart Entity Picker values
- Inline Entity Creation state
- existing save/list synchronization
- post-save Quote/Invoice detail routing

Repeated shortcut events are ignored to reduce accidental duplicate submissions.

## Table Integrations

Keyboard row navigation was added to:

- shared `EntityTable` for Companies, Contacts and Customers
- Quotes table
- Invoices table
- Payments table
- Opportunities pipeline selection

Companies, Contacts, Quotes, Invoices and Payments open their existing detail route with Enter. Customers currently support active-row and Space selection behavior but do not invent a missing standalone detail route.

## Contextual Actions

The contextual action strip now skips disabled actions during arrow navigation and supports Space activation for link-style action chips. Quote, Invoice, Payment, Customer, Company and Contact contextual actions keep native button/link semantics.

## PDF Preview Behavior

SPR-343B PDF actions remain intact:

- Preview, Download and Print are keyboard reachable.
- Escape closes the PDF preview.
- Closing the preview returns focus to the triggering action where possible.
- No automatic download or print behavior was added.

## Shortcut Help

`?` opens a compact help surface showing only implemented shortcuts. The help surface has an accessible title and description, closes with Escape or its close button and remains readable on mobile.

## Accessibility

- Focus states are visible on rows, action chips, dialogs and help controls.
- Native link, button and form semantics are preserved.
- Tab can still leave components normally.
- Disabled contextual actions are skipped by arrow movement.
- Editable fields keep native typing and cursor behavior.

## Mobile Behavior

All touch workflows remain unchanged. Shortcut hints are hidden or compact on narrow screens, and the help surface avoids horizontal overflow. Physical keyboards on tablets can use the same shortcuts.

## Import And Client Safety

- Generic keyboard platform does not import CRM or Sales modules.
- Client-sensitive code uses direct imports.
- No access to `window`, `document` or `navigator` happens during module initialization.
- Existing Command Center import graph remains intact.

## Known Limitations

- Shortcut customization and preferences are not implemented.
- Payments and Opportunities do not have standalone `Cmd/Ctrl + N` creation shortcuts until stable standalone dialogs exist.
- Customers do not open a detail route with Enter because no customer detail route exists yet.
- Full authenticated browser click-through depends on an active local authenticated session.
