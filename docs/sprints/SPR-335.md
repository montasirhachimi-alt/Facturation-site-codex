# SPR-335 — Professional Workspace Controls

Date: 2026-07-09

## Objective

Transform BOSIACO workspace controls into compact, polished productivity controls. This sprint improves the daily-use surfaces around search, filters, actions, table headers, selection states, pagination and empty states without changing business behavior.

## Scope

- UI/UX only.
- No Runtime, Services, APIs, Prisma, database, permission or business logic changes.
- No backend features and no new modules.
- Existing routes, filters, sorting, pagination and click behavior are preserved.

## Workspace Controls Improved

- Search bars now use tighter height, stronger text weight, clearer focus rings and consistent icon alignment.
- Filter panels use more compact spacing and select controls with a shared professional treatment.
- Primary actions now stand out through one reusable blue action style.
- Secondary actions and table actions are calmer and no longer compete with creation actions.
- Table headers are denser, clearer and more consistent across shared CRM tables and local Sales tables.
- Selected rows now have a stronger but controlled selected state with an inset accent and subtle ring.
- Bulk action surfaces are visually clearer when rows are selected.
- Empty states are more compact and helpful, with smaller illustration blocks and clearer CTAs.
- Pagination controls are tighter and better aligned for workspace use.

## Components Modified

- `EntityToolbar`
- `EntitySearchBar`
- `EntityFilterPanel`
- `FormField`

- `EntityTable`
- `EntityActionButton`
- `EntityActionMenu`
- `EntityBulkActions`
- `EntityPagination`
- Shared workspace control style constants for primary, secondary, table and select controls.

## Screens Affected

- CRM Companies
- CRM Contacts
- Sales Quotes
- Sales Invoices
- Sales Payments
- Any HR, Reports or future list workspace consuming the shared entity controls.

## Desktop Result

Desktop workspaces show controls with less visual friction, clearer primary actions and denser table headers. More useful rows and filters remain visible without making the interface feel cramped.

## Mobile Result

Controls keep clean stacking behavior, maintain tappable heights and avoid oversized empty/control blocks on narrow screens.
