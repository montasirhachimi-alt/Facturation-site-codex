# SPR-343 — Contextual Actions

Date: 2026-07-10

## Objective

Introduce a reusable Contextual Actions framework so BOSIACO can suggest the next logical action after opening a business entity or workspace.

## Scope

- Product-experience and client UI layer only.
- Existing routes, tabs and local handlers only.
- No Runtime, Services, APIs, Prisma, database, permissions, authentication or business logic changes.
- No AI, backend workflow automation, persistence changes or fake actions.

## Contextual Action Architecture

Added `src/platform/contextual-actions/` as a generic platform UI foundation:

- `contextual-action.types.ts` defines the declarative action model.
- `contextual-action.registry.ts` registers and returns available actions.
- `contextual-action.utils.ts` centralizes sorting and visual tone classes.
- `use-contextual-actions.ts` prepares sorted available actions for React surfaces.
- `contextual-action-strip.tsx` renders compact action chips with keyboard support.

The generic framework does not import CRM or Sales modules. Business pages pass already-resolved actions, routes and handlers into the strip.

## Action Model

Each action can define:

- `id`
- `entityType`
- `label`
- `ariaLabel`
- `description`
- `icon`
- `priority`
- `href`
- `onSelect`
- `available`
- `disabled`
- `disabledReason`
- `tone`

Actions remain declarative and are sorted by priority.

## Supported Entity Types

V1 supports contextual actions for:

- Quote
- Invoice
- Payment
- Customer workspace
- Company
- Contact

## Supported Actions

Quote:

- Create invoice when the quote is accepted and no invoice exists.
- Open linked invoice when already generated.
- Open linked company.
- Open pipeline when an opportunity exists.

Invoice:

- Record payment when a remaining balance exists.
- Open linked company.
- Open source quote when available.

Payment:

- Open linked invoice.
- Open linked company.

Customer workspace:

- Create customer.
- Open quotes.
- Open invoices.
- Open opportunities.

Company:

- Open contacts tab.
- Open opportunities tab.
- Open quotes tab.
- Open linked customers tab.

Contact:

- Open linked company.
- Open opportunities tab.
- Open quotes tab.
- Open meetings tab.
- Open notes tab.

## Integration Points

Contextual action strips were added to:

- Quote details workspace.
- Invoice details workspace.
- Payment details workspace.
- Customers workspace.
- Company details workspace.
- Contact details workspace.

Existing header metadata remains intact, while the contextual strip becomes the calm next-step area below the page identity.

## Keyboard Behavior

- Tab reaches every action chip.
- Enter activates focused links and buttons.
- Arrow Left / Arrow Up focuses the previous chip.
- Arrow Right / Arrow Down focuses the next chip.
- Disabled actions remain non-activatable and expose their reason through the title.

## Accessibility

- The strip uses a labeled `section` and `toolbar`.
- Every chip exposes an accessible label.
- Icons are decorative with `aria-hidden`.
- Focus rings remain visible.
- Link actions and button actions use the correct semantic element.

## Desktop Result

Entity pages now show a compact next-step strip directly under the header. The user can continue from quote to invoice, invoice to payment, payment to invoice, or CRM entity to relationship tabs without scanning the whole page.

## Mobile Result

Action chips wrap naturally, keep comfortable tap targets and avoid large button blocks. The strip stays compact above the primary workspace content.

## Known Limitations

- Actions are local/client-side suggestions only.
- No backend workflow, automation or AI recommendation is introduced.
- Actions such as duplicate, send or PDF download were not added where no stable existing workflow is available.
- Customer-specific actions are currently workspace-level because there is no dedicated customer detail route.
