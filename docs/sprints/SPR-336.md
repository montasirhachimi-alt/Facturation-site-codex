# SPR-336 — Professional Forms Experience

Date: 2026-07-09

## Objective

Make record creation and editing feel as premium as browsing BOSIACO workspaces. This sprint improves the form experience through shared dialog polish, field hierarchy, section grouping, consistent actions and calmer validation surfaces.

## Scope

- Product experience only.
- No Runtime, Services, APIs, Prisma, database, permission or business logic changes.
- No backend work and no new modules.
- Existing submit handlers, field values, routes and creation/editing behavior are preserved.

## Dialogs Improved

- Shared `EntityDialog` now uses a calmer overlay, softer modal radius, more professional shadow, compact header hierarchy and accessible dialog attributes.
- Error messages now use a clearer alert treatment with icon, border and readable line-height.
- Dialog sizing avoids oversized modal windows while still supporting professional multi-section forms.

## Shared Form Components Updated

- `FormField` now supports required indicators and helper text.
- `FormSection` groups related fields into compact, readable sections.
- `FormActions` standardizes Cancel and Save-style footer buttons.
- `FormBooleanField` gives checkboxes a clearer row treatment for relationship/role flags.
- Existing input styling remains compact from SPR-335 while gaining stronger hierarchy.

## Forms Reorganized

- Company creation now separates general identity, contact/address details and commercial qualification.
- Contact creation/editing now separates identity, contact channels and relationship context.
- Customer creation now separates customer identity from contact/qualification details.
- Quote creation preview now separates commercial context from financial conditions.

## Desktop Improvements

- Forms avoid one long flat grid and use responsive multi-column sections.
- Important identity fields appear first and are visually easier to scan.
- Footer actions are consistent and predictable across dialogs.

## Mobile Improvements

- Sections stack cleanly without horizontal scrolling.
- Inputs, checkboxes and footer actions retain comfortable tap targets.
- Dialogs remain compact enough for mobile while preserving readability.
