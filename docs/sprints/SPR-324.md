# SPR-324 — Product Experience Redesign (Phase 1)

## Summary

SPR-324 improves the perceived quality and usability of BOSIACO/HicoPilot without adding business features. The sprint focuses on Dashboard, Sidebar and shared CRM/Sales UI primitives so the product feels calmer, clearer and more premium.

## Objective

Reduce cognitive load, guide first-time users and make the main commercial experience feel like one coherent product.

## Scope

This sprint changes presentation only:

- Dashboard composition.
- Sidebar hierarchy and visual rhythm.
- Shared entity headers, cards, empty states, tables, filters, pagination, dialogs and form fields.
- Sales list empty states and table density.

No services, Runtime, Prisma, APIs, persistence, permissions or business workflows were redesigned.

## Product UX Decisions

### Dashboard

- Replaced the dense technical dashboard with a business cockpit.
- Started with `Bonjour <User> 👋` and the requested guidance copy: `Voici ce qui mérite votre attention aujourd'hui.`
- Reduced quick actions to five primary business actions: client, devis, facture, paiement and opportunité.
- Reduced KPIs to four executive signals: chiffre d'affaires, reste à encaisser, marge brute and pipeline actif.
- Removed chart-heavy sections from the first screen because they increased cognitive load before the user knew what to do next.
- Added `Priorités du jour` as a clear next-action area.
- Added compact recent activity to explain what happened without overwhelming the page.

### Sidebar

- Removed placeholder shortcut controls that added noise without active value.
- Reduced width and softened active states to make the navigation feel lighter.
- Added a short product explanation under the logo so the sidebar immediately communicates the product purpose.
- Kept the official business module navigation source unchanged.

### Empty States

- Upgraded shared empty states with a stronger illustration-like icon treatment.
- Added primary CTAs to Sales empty states so the user knows what to do next.
- Kept empty-state copy short and action-oriented.

### Tables

- Reduced row height slightly while keeping scanability.
- Softened hover states and borders.
- Improved table header rhythm and selected-row presentation through the shared `EntityTable`.
- Applied calmer table density to Sales Quotes, Invoices and Payments.

### Forms

- Improved shared form input contrast, focus rings and font weight.
- Made dialogs scroll safely on smaller screens.
- Preserved existing field groups and submit behavior.

### Colors

- Reduced decorative color usage in cards and actions.
- Kept blue primarily for primary actions, focus, links and product navigation.
- Left semantic status colors for badges where color communicates state.

### Typography

- Tightened header hierarchy and table headers.
- Kept display font for page titles and important values only.
- Reduced uppercase tracking where it made the UI feel overly technical.

### Consistency

- Shared UI primitives now make CRM, Sales, Pipeline, Quotes, Invoices, Payments and Dashboard feel closer to one product family.
- Existing CRM/Sales service and module boundaries remain intact.

### Responsive

- Dashboard sections use simpler responsive grids.
- Entity filters now adapt more naturally across tablet and desktop widths.
- Dialogs gain max-height scrolling for smaller viewports.

### Micro Interactions

- Hover states now use softer background shifts and shadow changes.
- Focus rings remain visible but calmer.
- Cards avoid unnecessary movement except where it reinforces clickability.

## Known Risks

- The product still has older ERP module pages outside the focused Dashboard/CRM/Sales scope.
- Some Sales table components still use local table markup and should later move to shared table primitives.
- Visual QA through a running browser was limited by sandbox port binding restrictions in the previous session.

## Release Notes

- Dashboard is now a welcoming business starting point.
- Sidebar is lighter and less noisy.
- Shared CRM/Sales UI primitives feel calmer and more premium.
- Sales empty states now guide the user with clear next actions.
