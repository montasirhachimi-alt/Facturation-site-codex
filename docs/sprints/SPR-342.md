# SPR-342 — Favorites & Recent

Date: 2026-07-10

## Objective

Add local Favorites and Recent items to the Command Center so frequently used destinations and records can be reopened without repeated search.

## Scope

- Command Center product-experience layer only.
- Local browser storage only.
- No Runtime, Services, APIs, Prisma, database, permissions, authentication or business logic changes.
- No backend persistence, synchronization, AI recommendations, folders or drag-and-drop ordering.

## Favorites Architecture

- Added a serializable `CommandCenterHistoryItem` model.
- Favorites are stored as stable snapshots, not React components or module objects.
- Favorite identifiers are deterministic:
  - navigation: `nav:/route`
  - record: `record:type:id`
- Favorite toggles are available from search results, Favorites and Recent sections.
- Toggling a favorite does not activate navigation.

## Recent Architecture

- Recent items are recorded when a user activates a navigable Command Center item.
- Recent entries are deduplicated by deterministic id.
- Reopened items move to the top.
- Recent storage is bounded to 10 items.
- Invalid or unparsable stored entries are ignored safely.

## Persistence Strategy

Browser storage keys:

- `bosiaco.command-center.favorites.v1`
- `bosiaco.command-center.recent.v1`

Storage is:

- client-only
- SSR-safe
- parse-failure safe
- bounded
- schema-checked at read time

If localStorage is unavailable, the Command Center still works without crashing.

## Supported Item Types

Favorites and Recent support the same navigable item types already exposed through Command Center search:

- Navigation destinations such as Dashboard, CRM, Sales, Products/Stock, Reports and Settings.
- Records exposed by Smart Global Search:
  - companies
  - contacts
  - customers
  - opportunities
  - quotes
  - invoices
  - payments

No new record adapters were added.

## Command Center Integration

When the search query is empty, the Command Center shows:

1. Favorites
2. Recent
3. Quick Create

When the user types, the Command Center preserves current search behavior:

1. Quick Create
2. Navigation
3. Records

This keeps the empty state focused on repeated access while preserving search ranking once the user types.

## Keyboard Behavior

- Arrow Down and Arrow Up continue moving through visible items.
- Enter activates the selected favorite, recent item, search result or quick create action.
- Esc closes the Command Center.
- Favorite toggles are keyboard-accessible with Enter or Space when focused.
- The active item index resets safely when sections appear or disappear.

## Accessibility

- Favorite toggles expose `aria-label` and `aria-pressed`.
- Favorites are indicated with both icon fill and accessible state, not color alone.
- Section headings and compact empty states remain visible.
- Row navigation and favorite toggling have separate activation behavior.

## Desktop Result

Frequently accessed pages and records are reachable immediately after opening the Command Center. The layout remains compact and avoids showing every section at maximum size while searching.

## Mobile Result

The same Command Center layout is used on mobile. Favorite toggles remain compact, rows avoid horizontal overflow and metadata stays inside the existing dialog width.

## Import-Safety Decisions

- Generic history types, storage and utilities do not import CRM or Sales modules.
- The Command Center provider imports only platform search history helpers.
- No generic UI component imports business modules.
- Existing direct import practices from SPR-341A are preserved.
- No broad UI barrel was modified to re-export client-sensitive history or picker data.

## Limitations

- Favorites and Recent are local to the browser.
- There is no account sync, workspace sync or backend persistence.
- Recent items are recorded reliably from Command Center activation only.
- If local demo data changes and a route becomes invalid, the stored entry is ignored only when read shape is invalid; route existence is still handled by current Next.js routing.
