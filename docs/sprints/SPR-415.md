# SPR-415 — Business Timeline Engine Foundation

## Summary

SPR-415 introduces the Business Timeline Engine foundation.

The sprint creates a generic Runtime-first timeline package capable of reconstructing an entity journey from registered providers. It does not implement Sales, CRM, Inventory, Accounting or HR providers yet, and it does not change existing business workflows.

## Architecture

New Runtime package:

```text
src/runtime/timeline/
  business-timeline.types.ts
  business-timeline.registry.ts
  business-timeline.service.ts
  business-timeline.utils.ts
  index.ts
```

The package defines:

- `TimelineEvent`
- `TimelineQuery`
- `TimelineProvider`
- `TimelineRegistry`
- `BusinessTimelineRuntime`

Runtime flow:

```text
TimelineProvider registrations
  ↓
TimelineRegistry
  ↓
BusinessTimelineRuntime.getTimeline()
  ↓
filtered, deduplicated, chronologically sorted TimelineEvent[]
```

Application service:

```text
src/services/timeline/TimelineService.ts
```

The service exposes a single orchestration point over the Runtime package.

Reusable UI:

```text
src/ui/timeline/
  BusinessTimeline.tsx
  TimelineCard.tsx
  TimelineEvent.tsx
  index.ts
```

The UI only renders generic `TimelineEvent[]`. It does not know about Quotes, Orders, Invoices, Payments, Reservations or any other business module.

## Validation

Required validation:

```bash
npm run typecheck
npm run build
npm run validate:runtime
```

SPR-415 adds focused runtime validation for:

- Timeline provider registration
- duplicate provider rejection
- provider lookup and unregister
- timeline event filtering by entity
- duplicate event removal by semantic event id
- newest-first chronological ordering
- immutable returned event lists
- Runtime import safety
- UI domain-agnostic rendering

## Future Extensions

Future sprints can add providers without changing the generic engine:

- `CRMTimelineProvider`
- `SalesTimelineProvider`
- `InventoryTimelineProvider`
- `AccountingTimelineProvider`
- `HRTimelineProvider`

Expected future integrations:

- Company journey
- Contact journey
- Quote journey
- Sales Order journey
- Delivery Note journey
- Inventory movement journey
- Accounting document journey

SPR-415 intentionally does not add database tables, APIs, notifications, activity feed behavior or business-module page changes.
